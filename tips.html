Tips

Every tip in the Tips List sidebar also links to the appropriate tip on this page.

Try using Ctrl+F (windows), Cmd+F (mac) or the Find function on mobile browsers to search for keywords related to something you're looking for.


Mapping

Edit only selected terrain

Sometimes you may try to edit terrain without affecting other individual terrain pieces around it. You can toggle the option to only paint the selected terrain, instead of any terrain the brush is on.

Object/selection proportional scaling

You can switch between Selection Center and Object Center when scaling multiple objects at once. Selection Scaling retains the relative distance between each selected object, and Object Scaling keeps each object in place.

Selecting weird wall segments

Sometimes when you want to click on a wall segment, you will click on an object behind it or a different part of the wall. This is because the physics model of the wall segment is different from the visual model, and the physics model is what you have to click on. You can select a wall, select the dropdown menu with the “Eye” icon, and choose physics to toggle seeing physics models.

Creating rooftops easily

Making a roof from scratch can be a pain, but a quick way is to duplicate a wall with Shift+Mouse Drag upwards, then change its root ID to the matching roof. This way you don't have to shape walls twice and try to line them up correctly, as they are already shaped and lined up correctly.

Randomizing instance painter settings

Changing the settings around in the Instance Painter to give you more variation can help create more interesting detail to your maps. You can change how fast it creates instances (flow), random rotation, and minimum and maximum sizes of created instances. AlignNormal is also a setting which can make instances align with the terrain.



Scripting

S_ Prefix for objects referred to in scripts

You can add the prefix S_ to the name of an object to make the engine treat it as an object important to scripting. This will make a warning prompt appear when you try to delete the object, saying it could break scripting if it is deleted. You should do this for any character, item, etc. that is or will be used in a script.

Search and autocomplete statements

After typing any word in the story editor, it will try to display every call, query, and event name that has that word in it. If you use the up and down arrow keys you can navigate to each statement, and it will show what fields are required for the highlighted statement. It also color codes events as Purple, queries as Red, and calls as Green. You can use this to search for statements and see how to utilize each one quickly.

Refresh new object names/IDs in story editor

If you just created new names for objects, they might not show up in the story editor to autocomplete if you type them out. This is because the engine hasn't refreshed its database of object names and IDs, so in order to do that, go to File > Generate Definitions in the story editor.

Quickly test scripting with character

You'll want to test your scripts out when you type them out, but this can be a pain especially if your scripting involves much later parts of a level. One way to do this is create a character and place them near where you will test your scripting, and then add them to the DB_IsPlayer database and make them a player character in a story script.

After compiling the script and reloading the level, your character will spawn at the level start, so just click them, press J to jump to their original spot, and move them back into place, then go to Game view and control them to test your level out.

Search for content across all script goals

You can use the Find Replace function found under the Edit tab in the story editor to search for content across every script goal found and inherited in your project. This is incredibly useful for times when you need to find where an object is being referenced in a script, or when searching through the Divinity Origins campaign to find where and how Larian Studios scripted certain events in Divinity: Original Sin II.



Stats

Search for content in stats lists

You can easily search for key words in stats lists with Ctrl+F. There are arrow buttons to navigate to the next instance of the searched word and previous instance, a button to change sensitive case and search for entire words, and a Find All button that lists every instance in the Find window.

Filter content in stats lists

You can add filters to exclude or include any content you don't or do want to see in a stats list. You can specify which column to target in a filter as well as what conditions it should satisfy, such as "Contains" or "Is greater than". For example, you can show only skills that have a cooldown of one turn, with Cooldown > Equals > 1.

Parenting: Re-using stat values for new entries

Make ample use of the Parent column in stats lists to give an entry the same stats as the entry you enter into the column, and then just modify what you want. This will save you a lot of time and is extremely helpful for making pretty much anything in the stats editor. For example, if I wanted to make a new jump skill, I would probably use Phoenix Dive as a parent and then make changes specific to my new skill.
